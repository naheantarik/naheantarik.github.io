Queue Management System

  • Live Demo
  • Source code

The purpose of this project is to develop a Queue Management System for organizing queuing system that can analyze the queue. This queuing architecture model can switch between different scheduling algorithms according to the testing result i.e. the average waiting time by using two different queue control systems, which have developed status and take decision which customer to be served first. The project includes admin and user sides. Admin side manages and views all the management like user details, services, token number, serial etc. And user can choose their services and also payment scheme. Based on their service time and payment scheme user will get their token number.

The Automated Queue Management System is design to manage certain customers with single department. This queuing architecture model can switch between different scheduling algorithms according to the testing result. Example the average waiting time by using two different queue control systems, which have developed status and take decision which customer to be served first.

Requirements

  • User can choose their required payment scheme.
  • When a user pays in full, he/her service will be paid first.
  • When a user pays in half, he/her service will be paid second.
  • If there is a free client, he will be receive his service in late.
  • User will be take their token number.
  • User will be check their serial.
